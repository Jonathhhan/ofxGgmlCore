# Documentation

Start here when you are new to the rewritten `ofxGgmlCore` line.

## First Reads

| Document | Use it for |
| --- | --- |
| `index.html` | GitHub Pages landing page for the addon family. |
| `QUICKSTART.md` | Clone, setup, and run the focused examples. |
| `EXAMPLES.md` | Choose between simple, text, chat, and embedding examples. |
| `RELEASE_NOTES.md` | See what changed in the current release line. |
| `../CONTRIBUTING.md` | Follow the change rules before editing core. |

## Design And Scope

| Document | Use it for |
| --- | --- |
| `CORE_CONTRACT.md` | Check what belongs in core and what must stay out. |
| `ARCHITECTURE.md` | Understand the current layer split and dependency direction. |
| `COMPANIONS.md` | Decide when work should become a companion addon. |
| `ROADMAP.md` | Track planned work and release rules. |

## Optional Areas

| Document | Use it for |
| --- | --- |
| `SEGMENTATION.md` | Understand the current segmentation/SAM3 boundary. |
| `RELEASE_READINESS.md` | Run the final checklist before tagging or widening the API. |

For normal usage, read `QUICKSTART.md` first. For implementation work, read
`CORE_CONTRACT.md` and `COMPANIONS.md` before adding new public API.
