ofxGgml addon family

A small openFrameworks addon family for local ggml-based AI workflows. Core stays narrow and dependable; domain workflows live in companion addons.

Baseline

Every active companion now has a tagged v1.0.1 baseline: README, docs, root-level examples, local validation scripts, release-candidate scripts, headless tests, public version metadata, and artifact hygiene. Core owns shared ggml setup and small backend-neutral primitives; companions own model-specific workflows.

Fresh Checkout

Clone the core addon into the openFrameworks addons folder first. Companion addons depend on it.

git clone https://github.com/Jonathhhan/ofxGgmlCore.git
cd ofxGgmlCore
scripts\first-run.bat
scripts\doctor.bat
scripts\run-simple-example.bat -Build

Design Rule

Move code into ofxGgmlCore only when it is backend-neutral, small, stable, and useful to most companion addons. Model-specific workflows, large examples, media assets, network retrieval, and product-style tools belong in companions.

Previous broad experiments live on the legacy-full branch. This page tracks the cleaner addon family around the rewritten main line.
